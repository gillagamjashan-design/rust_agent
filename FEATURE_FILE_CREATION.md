# Automatic File Creation Feature

## What It Does

Rusty now automatically creates Rust source files when you ask it to generate code. No manual copy-pasting required!

## Examples

### Create a Hello World Program

**You ask:** "Create a hello world program"

**Rusty responds with:**
```rust
fn main() {
    println!("Hello, world!");
}
```

**And automatically creates:**
- `src/main.rs` with the code

### Start a New Project

**You ask:** "Make a new Rust project called myapp"

**Rusty automatically creates:**
- `Cargo.toml` with `name = "myapp"`
- `src/main.rs` with a basic main function

### Generate a Struct

**You ask:** "Create a User struct with name and email fields"

**Rusty automatically creates:**
- `src/user.rs` with the struct definition

## How It Works

1. **Detection**: Rusty detects when you're asking for code generation
2. **Generation**: Claude API generates the code
3. **Extraction**: Code blocks are extracted from the response
4. **Creation**: Files are created automatically in your working directory
5. **Notification**: You see a summary of what was created

## Trigger Keywords

File creation happens automatically when your query contains:
- "create"
- "make"
- "generate"
- "write"
- "add"
- "new project"
- "hello world"
- And similar phrases

## Smart Filename Detection

Rusty infers the correct filename based on:
- **Comments in code**: `// src/config.rs`
- **Main function**: `fn main()` → `src/main.rs`
- **Tests**: `#[test]` → `tests/integration_test.rs`
- **Structs/Enums**: `struct User` → `src/user.rs`
- **Cargo files**: `[package]` → `Cargo.toml`

## Append Mode

If a file already exists, Rusty appends the new code with a separator:

```rust
// Existing code...

// --- Auto-generated by Rusty agent ---

// New code...
```

This prevents accidental overwrites!

## Supported File Types

Currently supports:
- ✅ Rust source files (`.rs`)
- ✅ Cargo configuration (`.toml`)
- ✅ More file types coming soon!

## Safety Features

- **No Overwrites**: Existing files are never replaced
- **Clear Markers**: Auto-generated content is clearly labeled
- **Directory Creation**: Parent directories are created automatically
- **Non-Blocking**: File creation doesn't freeze the GUI
- **Error Handling**: Failed file creations don't break your chat

## Knowledge Templates

Rusty uses 6 built-in templates:

1. **main.rs** - Hello world and binary programs
2. **Cargo.toml** - Project configuration
3. **lib.rs** - Library crates with tests
4. **integration_test.rs** - Test files
5. **module.rs** - Struct/enum modules
6. **mod.rs** - Module directory structures
